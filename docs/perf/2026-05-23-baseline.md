## ⚠️ Config caveat: the Workstream-A numbers above are NOT the app's config

`scripts/profile_pipeline.py` constructs `Pose2D(device="cpu", mode="lightweight")`,
so every table above measures **lightweight + CPU**. The app (and `test_2D_3D.py`)
run `Pose2D()` = **balanced (YOLOX-m + RTMPose-m) + CoreML** by default. The real
config measured directly on the fixture (M5 Max, balanced+CoreML):

- `pose.infer` (detect + pose + decode): **~20.7 ms mean / 25.3 ms p95** → ~48 fps single-thread ceiling, **~143% CPU** (≈7.9% of 18 cores).
- `put_thai_text` ×2 (a HOLD frame's overlays): **~3.76 ms** — never exercised by the harness (it mimics canvas compose with `cv2.line`/`rectangle` only).

When re-baselining, parametrize `profile_pipeline.py` with the real `mode` /
`accelerator` and include `put_thai_text` + the actual `screens.*` draws.


## Workstream B (post-review, 2026-05-23): per-frame waste + startup

Three changes, all visualization-preserving (3D rig / measurement rate unchanged):

1. **Inference dedup + 30 fps loop cap** (`app.py`, `capture.read_latest_with_ts`). The display loop ran faster than the camera, re-inferring ~25–33% duplicate frames. Now inference + 3D-buffer push are gated on a new capture timestamp; the loop is capped at 30 fps. `test_2D_3D.py` uses the same gate.
2. **Thai-text sprite cache** (`render.put_thai_text` / `_thai_sprite`). Replaced the per-call full-frame `BGR→RGB→PIL→RGB→BGR` round-trip with a cached RGBA sprite + numpy alpha-composite. Pixel-equivalent to the old direct draw (≤1 LSB on <0.3% of pixels).
3. **CoreML model cache** (`pose2d._COREML_PROVIDER`): `ModelCacheDirectory` + `SpecializationStrategy=FastPrediction`.

End-to-end, simulated 30 fps camera driving the real loop (balanced+CoreML, M5 Max):

| Metric | Before (original) | After (B1+B2+B3) | Change |
|---|---:|---:|---|
| Pose inferences / sec | 44.0 | 27.7 | −37% |
| Display-loop iterations / sec | 44.0 | 27.7 | −37% |
| Process CPU (% of one core) | 150.8% | 77.5% | **−49%** |
| Process CPU (% of 18-core) | 8.4% | 4.3% | −49% |
| Thai text / HOLD frame (warm) | 3.76 ms | 0.097 ms | −97% (≈39×) |
| Thai text (cold rebuild, ~every 3 s) | 3.76 ms | 2.63 ms | one-off on text change |
| Pose2D construction (warm cache) | 0.5 s | 0.1 s | −80% |
| Useful unique frames / sec · 3D lift | 30 · 6 Hz | 30 · 6 Hz | unchanged |

Process CPU roughly halved with identical on-screen output and measurement rate.
